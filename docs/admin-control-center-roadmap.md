# Admin Control Center Roadmap

## Цель
Создать отдельный контур управления платформой на адресе `/admin` для мониторинга, модерации и операционного управления.

## Что уже реализовано
- Отдельный адрес админки: `/admin`
- Role-based доступ (только `role=admin`)
- Сводная аналитика: пользователи, каналы, офферы, матчи, сообщения, отзывы
- Распределения по статусам и топ-ниши
- Ленты последних пользователей, обменов и сообщений

## Следующие модули (приоритет)
1. Пользователи
- Поиск/фильтр по роли, дате, активности
- Просмотр карточки пользователя и связанных каналов
- Смена роли `user/admin`
- Блокировка/разблокировка с причиной

2. Каналы и модерация
- Флаги каналов и журнал решений модератора
- Массовые действия: верификация, деактивация, пометка риска
- Аномалии: резкий рост/падение метрик, подозрительные паттерны

3. Обмены и сделки
- Граф воронки: offer -> match -> accepted -> completed -> reviewed
- Поиск всех связей «кто с кем обменивался»
- Таймлайн сделки + участники + чат + отзывы

4. Чаты и поддержка
- Глобальный монитор чатов (поиск по ключевым словам)
- Флаги токсичности/спама
- Эскалация в тикет поддержки

5. Финансы/монетизация (если будет)
- Платежные события, статусы подписок, возвраты
- MRR/Churn/ARPU

6. Надежность и безопасность
- Security events dashboard
- Rate-limit incidents
- Админ-аудит всех критичных действий

## Технические принципы
- Все мутации только в транзакциях
- Каждое критичное действие писать в `action_logs`
- Ролевой доступ на backend и frontend
- Пагинация и индексы для тяжелых выборок
- Дашборд должен собираться <= 1.5s на средних данных

## KPI для админки
- Time-to-detect инцидента
- Time-to-resolve тикета
- Доля успешных обменов
- Доля повторных обменов
- Средняя оценка после обмена
